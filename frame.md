# PDF2Chunk 项目框架设计

## 1. 项目概述

**项目名称**: pdf2chunk  
**项目定位**: RAG构建数据库的前置项目——将扫描件PDF转化为一题一Chunk  
**核心目标**: 将包含数学题目的扫描件PDF，预处理成结构化的文本格式（数学公式用LaTeX表示），并按题目分割成独立Chunk，为后续Embedding和RAG检索做准备。

---

## 2. 核心需求分析

### 2.1 输入
- 扫描件PDF（高数题库）
- 每页包含多个题目
- 题目可能跨页
- 包含数学符号、公式

### 2.2 输出
- 结构化Chunk列表（约1000个Chunk）
- 每个Chunk包含：
  - 题目原文（LaTeX格式数学公式）
  - 自动标注的元数据：难点、知识点、题型

### 2.3 关键技术挑战
1. **无用信息排除**: 过滤章节名、书名、页眉页脚等非题目内容
2. **数学公式识别**: 精确识别并转换为LaTeX格式
3. **题目分割**: 单页多题的正确切分
4. **跨页合并**: 跨页题目的正确合并

## 4. 模块详细设计

### 4.1 模块一：PDF图像提取与预处理

**功能**: 将PDF转换为图像，并进行预处理以提升OCR质量

**输入**: PDF文件路径  
**输出**: 预处理后的页面图像列表

**处理步骤**:
1. PDF逐页渲染为图像（DPI建议300+）
2. 图像预处理：
   - 灰度化
   - 去噪（高斯滤波/中值滤波）
   - 纠偏（检测文本方向并旋转校正）
   - 二值化（自适应阈值）
   - 边框/页眉页脚裁剪

**配置项**:
```yaml
pdf_to_image:
  dpi: 300
  fmt: "png"

preprocess:
  denoise: true
  deskew: true
  binarize: true
  crop_margin: [50, 50, 50, 50]  # 上下左右裁剪像素
```

---

### 4.2 模块二：OCR文字识别与公式提取

**功能**: 识别图像中的文字和数学公式，公式转换为LaTeX

**输入**: 预处理后的页面图像  
**输出**: 带LaTeX公式的原始文本

**实现方案**:
- 使用支持公式识别的OCR服务/模型
- 识别结果包含：普通文本 + LaTeX格式的数学公式

**输出格式示例**:
```
设函数 $f(x) = \int_{0}^{x} \frac{\sin t}{\pi - t} dt$，计算 $\int_{0}^{\pi} f(x) dx$。
```

**配置项**:
```yaml
ocr:
  provider: "glm-4v-flash"  # 从.env读取
  api_key: "${ZHIPUAI_API_KEY}"
  prompt: |
    请识别图片中的数学题目内容，要求：
    1. 所有数学公式必须使用LaTeX格式表示
    2. 保留题目的完整内容
    3. 忽略页眉、页脚、章节标题等非题目内容
    4. 输出纯文本，不要添加任何解释
```

---

### 4.3 模块三：无用信息过滤

**功能**: 识别并过滤非题目内容（章节名、书名、页眉页脚等）

**输入**: OCR原始文本  
**输出**: 过滤后的题目相关文本

**过滤策略**:
1. **规则过滤**:
   - 匹配常见章节模式：第X章、第X节、Exercise X.X
   - 匹配页眉页脚：页码、书名重复出现
   - 匹配说明性文字："本章小结"、"参考答案"等

2. **启发式过滤**:
   - 文本长度异常（过短可能是标题）
   - 位置信息（页面顶部/底部的固定区域）
   - 字体大小差异（标题通常更大）

3. **LLM辅助过滤**:
   - 对疑似非题目内容，使用LLM判断

**配置项**:
```yaml
filter:
  rules:
    - pattern: "^第[一二三四五六七八九十\\d]+章"
    - pattern: "^\\d+\.\\d+.*题"
    - pattern: "^(练习|习题|复习题)"
  min_content_length: 20
  use_llm_filter: true
```

---

### 4.4 模块四：题目分割与跨页合并

**功能**: 将连续文本按题目切分，处理跨页题目

**输入**: 过滤后的页面文本序列  
**输出**: 独立的题目Chunk列表

**分割策略**:

1. **题号识别**:
   - 识别题目编号模式：1. 、1、（1）、【1】等
   - 支持阿拉伯数字和中文数字

2. **单页多题分割**:
   - 根据题号位置切分
   - 处理子题（如 1.(1)、1.(2)）

3. **跨页合并**:
   - 检测题目在页面末尾是否完整
   - 使用启发式规则判断：
     - 文本是否以标点符号结尾
     - 是否包含完整的数学表达式
     - 下一页开头是否以题号开始
   - 不完整题目与下一页开头内容合并

**输出格式**:
```json
{
  "chunks": [
    {
      "id": "chunk_001",
      "page_range": [1, 1],
      "question_number": "1",
      "content": "设函数 $f(x) = x^2$，求 $f'(x)$。",
      "is_complete": true
    },
    {
      "id": "chunk_002",
      "page_range": [1, 2],
      "question_number": "2",
      "content": "计算定积分 $\\int_{0}^{1} x^2 dx$ 的值...",
      "is_complete": true
    }
  ]
}
```

**配置项**:
```yaml
segmentation:
  question_patterns:
    - "^\\s*\\d+[\\.、]\\s*"
    - "^\\s*[（(]\\d+[)）]\\s*"
    - "^\\s*[第]?[一二三四五六七八九十]+题"
  cross_page_merge:
    enabled: true
    incomplete_indicators:
      - "未以标点结尾"
      - "数学公式未闭合"
      - "下一页以题号开头"
```

---

### 4.5 模块五：LLM自动标注

**功能**: 使用LLM对每道题目进行自动标注

**输入**: 题目Chunk  
**输出**: 带元数据的题目Chunk

**标注维度**:
1. **知识点**: 题目涉及的具体知识点（如"定积分计算"、"导数应用"）
2. **难度等级**: 简单/中等/困难
3. **题型**: 选择题/填空题/计算题/证明题/应用题
4. **解题方法**: 题目主要使用的解题方法

**Prompt设计**:
```
请分析以下高数题目，输出JSON格式的标注结果：

题目：{question_content}

要求输出：
{
  "difficulty": "简单|中等|困难",
  "question_type": "单选题|多选题|填空题|计算题|证明题|应用题",
  "knowledge_points": ["知识点1", "知识点2"]
}
```

**配置项**:
```yaml
annotation:
  provider: "deepseek-chat"  # 从.env读取
  api_key: "${DEEPSEEK_API_KEY}"
  model: "${DEEPSEEK_MODEL}"
  batch_size: 10
  retry_times: 3
```

## 6. 配置管理

### 6.1 环境变量 (.env)

```bash
# OCR服务配置
ZHIPUAI_API_KEY=your_key_here
GLM_MODEL=glm-4v-flash

# LLM标注配置
DEEPSEEK_API_KEY=your_key_here
DEEPSEEK_MODEL=deepseek-chat
```

### 6.2 配置文件 (config.yaml)

```yaml
# PDF处理配置
pdf:
  dpi: 300
  batch_size: 10

# OCR配置
ocr:
  provider: "zhipu"
  model: "${GLM_MODEL}"
  max_retries: 3
  timeout: 60

# 分割配置
segmentation:
  question_patterns:
    - "^\\s*\\d+[\\.、]"
  enable_cross_page_merge: true

# 标注配置
annotation:
  provider: "deepseek"
  model: "${DEEPSEEK_MODEL}"
  batch_size: 5
```

## 8. 质量保障

### 8.1 验证机制

1. **OCR质量检查**:
   - 置信度阈值过滤
   - 人工抽检机制

2. **分割正确性验证**:
   - 题号连续性检查
   - 内容完整性检查

3. **标注一致性检查**:
   - 相同知识点标注一致性
   - 难度分布合理性

### 8.2 异常处理

| 异常场景 | 处理策略 |
|---------|---------|
| OCR识别失败 | 记录错误，跳过该页，继续处理 |
| 公式识别不完整 | 标记待人工审核 |
| 跨页合并异常 | 使用启发式规则，记录可疑Chunk |
| LLM标注失败 | 重试3次，失败后标记为未标注 |

---

## 9. 性能考虑

1. **并发处理**: PDF页面可并行OCR识别
2. **缓存机制**: OCR结果本地缓存，避免重复调用
3. **批量标注**: LLM标注采用批量请求，减少API调用次数
4. **断点续传**: 支持中断后从上次位置继续处理

---

## 10. 扩展性设计

1. **OCR服务可替换**: 通过配置切换不同OCR提供商
2. **标注维度可扩展**: 元数据结构支持新增标注字段
3. **多科目支持**: 通过配置不同的Prompt和标注维度支持其他科目
